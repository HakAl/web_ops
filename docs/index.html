The Skills Team

We build AI teams that ship better code

AI teams that plan, critique, build, and validate

Most AI coding tools work alone. One model, one voice, one blind spot. We tried something different: we built a team.

The Skills Team is an open-source project that gives your AI coding agent a crew of personas — a planner, a critic, a builder, a reviewer — that collaborate in shared context. They argue with each other, catch bad ideas early, and ship code that's been challenged before you even look at it.

This blog is where we write about what we're learning: the workflows that work, the failures that taught us something, and the patterns we keep coming back to.

The Pattern

Every team we build converges on the same structure: Plan → Critique → Build → Validate.

Personas, Not Processes

The team members are personas in the same context, not isolated agents. They can hear each other, interrupt, and collaborate in real-time. Same context beats parallel agents — every time.

Self-Improving Protocols

The team writes their own TEAM.md. They run retrospectives. They update their own skills based on what they learn. You're the founder who steps back and lets them evolve.

Safety Rails

  • No Lobotomies — Cannot edit IMMUTABLE sections of any skill
  • Guardian's Law — Every team has a guardian; all self-modifications pass validation
  • Stay in Your Lane — Teams only modify their own directories
  • User Has Final Say — All commits visible; user controls merges and pushes

Team-as-a-Template

The pattern isn't locked to one domain. Fork it, rename the personas, point it at your problem. Three teams run on it today.

Engineering

Plan → Critique → Build → Validate

The original team. Seven personas that design, challenge, implement, and review code.

Web Ops

Design → Critique → Build → Review

Site strategy, content, frontend, analytics. Same pattern, different domain.

QA & Compliance

Strategy → Audit → Test → Validate

Testing strategy, security audits, cross-platform validation. Finds the bugs you shipped.

Each team has its own guardian, its own protocols, and its own TEAM.md. The pattern is the constant — the personas adapt to the work.

Tested on Claude Code, Codex, and cross-platform. The genesis prompt works everywhere.

Get Started

Quick Install:

curl -sL https://raw.githubusercontent.com/HakAl/team_skills/master/install.sh | bash

Works on Linux, macOS, and Windows (Git Bash). Updates with the same command.

  1. Run /team genesis to bootstrap your team's protocols
  2. Give them a task: /team "add input validation"
  3. Watch them plan, critique, build, and validate

Want multiple teams? Install once, create new directories, run /team genesis in each. The dispatch protocol handles cross-team communication.
